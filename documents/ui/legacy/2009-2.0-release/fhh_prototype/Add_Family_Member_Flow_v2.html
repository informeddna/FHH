PROPOSED FHH PROTOTYPE FLOW - ADD FAMILY MEMBER (v2)

USE CASE FLOW:
1. For a user creating a health history from scratch, the user begins by clicking "Add Family Member" on the My Family Health History page.
2. The system displays the submodal with a form to add all immediate family members, including siblings, children, aunts, uncles, and grandparents of the proband.
3. The user enters the number for each family member and indicates whether or not to add grandparents and clicks Save Family Members.
4. Ths system displays the My Family Health History page with an updated tree navigation showing the newly added family members.
5. The user may opt to Add Family Member again.
6. The system displays the submodal with a form to add an idividual family member.
7. The user selects the family member they wish to add (based on that person's relationship to the user/proband). 
          -- IF the user selects a family member other than Brother, Sister, Son, or Daughter the system displays fields to determine the specific relationship to the user/proband, such as Side of Family and/or which family member this person is directly related to from those they have already entered into their history.
8. The user may either Save & Add Another or Save & Continue.  IF Save & Continue...
9. The system displays the submodal to edit the selected family member's personal profile & health history.

PROS & CONS

PROS:
  -- connections between family members is forced.  no "floaters" allowed.
  -- simple, straight-forward entry forms.
  -- allows for quick entry of immediate family members if only the proband, mother & father exist.
  -- allows for quick entry of multiple family members with "save & add another".
  -- allows users to declare total number of immediate family members (siblings, children, grand parents), thus helping prevent gaps within the overall family history.
  -- allows user to add a family members with a designation of "history unknown", which would prevent gaps in the overall family history.  ** Does this conflict with any data requirements, such as Living/Deceased?
  -- 3 clicks from start to finish with the new family member's personal profile & health history entered.

CONS:
  -- Does not allow user to return to enter multiple family members at one time.
